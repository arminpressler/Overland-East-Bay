üß™ API Testing Workflow

Follow these steps to create a test trip and see how the modules update with live API data:

Step 1: Create a Trip Draft

Go to api.html and:

  1. Scroll to the Trips section
  2. Find POST /trips
  3. Click "Create Trip Draft"
  4. Copy the tripId from the response

Step 2: Update Trip Details

In api.html:

  1. Find PATCH /trips/{tripId}
  2. Paste your tripId in the Trip ID field
  3. Modify the JSON body with your trip details
  4. Click "Update Trip"

Tip: The default JSON includes all fields needed for the modules to display properly.

Step 3: Publish the Trip

In api.html:

  1. Find POST /trips/{tripId}/publish
  2. Paste your tripId
  3. Click "Publish Trip"

Step 4: Update Modules to Use Live Data

Enter your trip ID to update all modules to use live API data:

Or manually update: Change the data-api-url attributes below to: 

data-api-url="/api/trips/YOUR_TRIP_ID"

Module Usage

To use these modules on a trip page, include the CSS files in the <head> and the JavaScript files before </body>, then add the module HTML with appropriate data-trip-id and data-api-url attributes.

Example HTML

<!-- Trip Info Module -->
<div class="trip-info" 
     data-trip-id="TRIP_2026_02_12_Death_Valley_Basecamp"
     data-api-url="/api/trips/TRIP_2026_02_12_Death_Valley_Basecamp">
</div>

<!-- Trip RSVP Module -->
<div class="rsvp-container" id="rsvp-content">
    <!-- Content injected via JS -->
</div>

<!-- Trip Artifacts Module -->
<div class="trip-artifacts" 
     data-trip-id="TRIP_2026_02_12_Death_Valley_Basecamp"
     data-api-url="/api/trips/TRIP_2026_02_12_Death_Valley_Basecamp">
</div>
